# 1. SETUP INICIAL (FASE 0) <a name="fase-0"></a>

## 1.1 Registrar Aplicação no ML Developers

1. Acessar: https://developers.mercadolivre.com.br/
2. Criar aplicação com scopes: `read`, `write`, `offline_access`
3. Redirect URI: `https://seudominio.com/callback`
4. Tópicos de notificação: `items`, `orders_v2`, `shipments`, `items_prices`, `catalog_item_competition_status`
5. Anotar: `APP_ID` e `SECRET_KEY`

**Uma aplicação para as 4 contas.** Cada conta gera seu próprio par de tokens.

## 1.2 Gerar Tokens OAuth para as 4 Contas

Para cada conta ML, logado nela:

```
# Passo 1: Abrir no navegador
https://auth.mercadolivre.com.br/authorization?response_type=code&client_id=$APP_ID&redirect_uri=$REDIRECT_URI

# Passo 2: Trocar code por tokens
POST https://api.mercadolibre.com/oauth/token
{
  "grant_type": "authorization_code",
  "client_id": $APP_ID,
  "client_secret": $SECRET_KEY,
  "code": "TG-XXXXXXXX",
  "redirect_uri": $REDIRECT_URI
}

# Resposta:
{
  "access_token": "APP_USR-xxxxx",    // expira em 6h
  "refresh_token": "TG-xxxxx",         // válido 6 meses, uso único
  "user_id": 123456789
}
```

# 3. BANCO DE DADOS <a name="banco-de-dados"></a>

```sql
-- ==========================================
-- CONTAS
-- ==========================================
CREATE TABLE accounts (
    id SERIAL PRIMARY KEY,
    user_id BIGINT UNIQUE NOT NULL,
    name VARCHAR(100) NOT NULL,
    access_token TEXT,
    refresh_token TEXT,
    token_expires_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- ==========================================
-- SNAPSHOT DIÁRIO DA REPUTAÇÃO DA CONTA
-- ==========================================
CREATE TABLE account_health (
    id SERIAL PRIMARY KEY,
    account_id INTEGER REFERENCES accounts(id),
    snapshot_date DATE NOT NULL,
    -- Termômetro
    level_id VARCHAR(20),                -- '5_green', '4_light_green', '3_yellow', '2_orange', '1_red'
    power_seller_status VARCHAR(20),     -- 'platinum', 'gold', 'silver', null
    -- Métricas (60 dias)
    sales_completed INTEGER,
    claims_rate DECIMAL(6,4),
    claims_value INTEGER,
    cancellations_rate DECIMAL(6,4),
    cancellations_value INTEGER,
    delayed_rate DECIMAL(6,4),
    delayed_value INTEGER,
    -- Dados reais se protegido
    real_level VARCHAR(20),
    real_claims_rate DECIMAL(6,4),
    real_cancellations_rate DECIMAL(6,4),
    real_delayed_rate DECIMAL(6,4),
    -- Transações históricas
    total_transactions INTEGER,
    completed_transactions INTEGER,
    canceled_transactions INTEGER,
    positive_ratings DECIMAL(4,2),
    negative_ratings DECIMAL(4,2),
    neutral_ratings DECIMAL(4,2),
    -- Tags do vendedor
    seller_tags JSONB,
    -- Timestamps
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(account_id, snapshot_date)
);

-- ==========================================
-- ITENS (tabela principal)
-- ==========================================
CREATE TABLE items (
    id SERIAL PRIMARY KEY,
    account_id INTEGER REFERENCES accounts(id),
    item_id VARCHAR(20) UNIQUE NOT NULL,     -- 'MLB1234567'
    user_product_id VARCHAR(30),              -- 'MLBU...'
    title VARCHAR(500),
    sku VARCHAR(100),                          -- Vem de seller_custom_field OU atributo SELLER_SKU
    category_id VARCHAR(20),
    category_path VARCHAR(500),
    domain_id VARCHAR(100),
    price DECIMAL(12,2),
    base_price DECIMAL(12,2),
    original_price DECIMAL(12,2),
    currency VARCHAR(5),
    available_quantity INTEGER,
    sold_quantity INTEGER,
    initial_quantity INTEGER,
    condition VARCHAR(20),
    listing_type VARCHAR(30),                 -- 'gold_special', 'gold_pro', etc.
    status VARCHAR(20),                       -- 'active', 'paused', 'closed'
    sub_status VARCHAR(50),
    permalink TEXT,
    thumbnail TEXT,
    video_id VARCHAR(50),
    -- Envio
    shipping_mode VARCHAR(20),
    logistic_type VARCHAR(30),                -- 'fulfillment', 'xd_drop_off', 'cross_docking'
    free_shipping BOOLEAN,
    shipping_tags JSONB,
    -- Catálogo
    catalog_product_id VARCHAR(30),
    catalog_listing BOOLEAN,
    -- Canais
    channels JSONB,                           -- ['marketplace', 'mshops']
    -- Tags do anúncio
    tags JSONB,
    -- Custos
    freight_cost DECIMAL(10,2),                -- Vem de GET /orders (shipping_cost), NÃO do GET /items
    commission_rate DECIMAL(6,4),
    commission_value DECIMAL(10,2),
    fixed_fee DECIMAL(10,2),
    -- Datas
    date_created TIMESTAMPTZ,
    last_updated TIMESTAMPTZ,
    synced_at TIMESTAMPTZ DEFAULT NOW()
);

-- ==========================================
-- SCORE DE QUALIDADE POR ITEM (histórico)
-- ==========================================
CREATE TABLE item_performance (
    id SERIAL PRIMARY KEY,
    item_id VARCHAR(20) NOT NULL,
    snapshot_date DATE NOT NULL,
    -- Score /item/$ITEM_ID/performance
    score INTEGER,                            -- 0 a 100
    level VARCHAR(20),                        -- 'Basic', 'Standard', 'Professional'
    -- Buckets detalhados
    buckets JSONB,                            -- Lista completa de buckets/variables/rules
    pending_actions JSONB,                    -- Ações pendentes simplificadas
    -- Score /items/$ITEM_ID/health (legado)
    health DECIMAL(4,2),
    health_actions JSONB,
    -- Timestamps
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(item_id, snapshot_date)
);

-- ==========================================
-- EXPERIÊNCIA DE COMPRA POR ITEM
-- ==========================================
CREATE TABLE item_experience (
    id SERIAL PRIMARY KEY,
    item_id VARCHAR(20) NOT NULL,
    snapshot_date DATE NOT NULL,
    -- Reputação do item
    health_gauge VARCHAR(20),                 -- 'healthy', 'warning', 'unhealthy'
    reputation_color VARCHAR(20),
    reputation_value INTEGER,                 -- 0 a 100
    -- Problemas detalhados
    problems JSONB,
    total_problems INTEGER,
    main_problem_key VARCHAR(50),
    -- Vendas nos últimos 180 dias
    sales_180d INTEGER,
    -- Timestamps
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(item_id, snapshot_date)
);

-- ==========================================
-- FICHA TÉCNICA (gap de atributos)
-- ==========================================
CREATE TABLE item_specs (
    id SERIAL PRIMARY KEY,
    item_id VARCHAR(20) NOT NULL,
    snapshot_date DATE NOT NULL,
    category_id VARCHAR(20),
    -- Contadores
    total_fields INTEGER,
    filled_fields INTEGER,
    empty_fields INTEGER,
    -- Por tipo
    required_total INTEGER,
    required_filled INTEGER,
    required_empty INTEGER,
    hidden_total INTEGER,
    hidden_filled INTEGER,
    hidden_empty INTEGER,
    common_total INTEGER,
    common_filled INTEGER,
    common_empty INTEGER,
    relevant_total INTEGER,
    relevant_filled INTEGER,
    relevant_empty INTEGER,
    -- Score calculado (igual planilha EP)
    score_max DECIMAL(8,2),
    score_actual DECIMAL(8,2),
    score_percent DECIMAL(5,2),
    -- Detalhes
    missing_attributes JSONB,                 -- Lista de atributos faltantes com tipo e relevância
    -- Compatibilidades de veículos (AUTOPEÇAS)
    compat_status VARCHAR(30),                -- 'complete', 'incomplete', 'pending_suggestions', 'universal', null
    compat_total INTEGER,                     -- Total de veículos compatíveis cadastrados
    compat_pending_tags JSONB,                -- Tags: incomplete_compatibilities, pending_compatibilities, etc.
    compat_suggestions_count INTEGER,         -- Quantas sugestões do ML disponíveis
    -- Timestamps
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(item_id, snapshot_date)
);

-- ==========================================
-- VENDAS E CONVERSÃO (D-30/60/90/120)
-- ==========================================
CREATE TABLE item_sales (
    id SERIAL PRIMARY KEY,
    item_id VARCHAR(20) NOT NULL,
    snapshot_date DATE NOT NULL,
    -- Visitas
    total_visits INTEGER,
    -- Vendas por período
    sales_d30 INTEGER,
    sales_d60 INTEGER,
    sales_d90 INTEGER,
    sales_d120 INTEGER,
    -- Faturamento por período
    revenue_d30 DECIMAL(12,2),
    revenue_d60 DECIMAL(12,2),
    revenue_d90 DECIMAL(12,2),
    revenue_d120 DECIMAL(12,2),
    -- Conversão
    conversion_rate DECIMAL(6,4),
    -- Cancelamentos e devoluções
    cancel_rate DECIMAL(6,4),
    return_rate DECIMAL(6,4),                  -- Devoluções / vendas totais do item
    -- Avaliações
    total_reviews INTEGER,
    avg_rating DECIMAL(3,2),
    rating_1 INTEGER,
    rating_2 INTEGER,
    rating_3 INTEGER,
    rating_4 INTEGER,
    rating_5 INTEGER,
    -- Curva ABC
    abc_revenue VARCHAR(1),                   -- 'A', 'B', 'C'
    abc_profit VARCHAR(1),
    -- Timestamps
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(item_id, snapshot_date)
);

-- ==========================================
-- MÉTRICAS DE ADS
-- ==========================================
CREATE TABLE ads_metrics (
    id SERIAL PRIMARY KEY,
    account_id INTEGER REFERENCES accounts(id),
    item_id VARCHAR(20),
    campaign_id VARCHAR(30),
    metric_date DATE NOT NULL,
    impressions INTEGER,
    clicks INTEGER,
    ctr DECIMAL(6,4),
    cost DECIMAL(12,2),
    cpc DECIMAL(8,2),
    direct_units INTEGER,
    direct_amount DECIMAL(12,2),
    organic_units INTEGER,
    organic_amount DECIMAL(12,2),
    roas DECIMAL(8,2),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(account_id, item_id, campaign_id, metric_date)
);

-- ==========================================
-- ALERTAS
-- ==========================================
CREATE TABLE alerts (
    id SERIAL PRIMARY KEY,
    account_id INTEGER REFERENCES accounts(id),
    item_id VARCHAR(20),
    alert_type VARCHAR(50),
    -- Tipos: 'reputation_risk', 'unhealthy_item', 'catalog_forewarning',
    -- 'missing_pi', 'stock_rupture', 'negative_tag', 'not_market_price',
    -- 'spec_gap', 'conversion_drop', 'sales_drop', 'ads_roas_low',
    -- 'freight_disadvantage', 'moderation_penalty', 'price_above_suggestion',
    -- 'compat_incomplete', 'compat_pending', 'compat_position_missing'
    severity VARCHAR(10),                     -- 'red', 'yellow', 'info'
    title VARCHAR(200),
    message TEXT,
    action_required TEXT,
    data JSONB,                               -- Dados extras do alerta
    resolved BOOLEAN DEFAULT FALSE,
    resolved_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- ==========================================
-- COMPETIÇÃO DE PREÇO (SUGESTÕES ML)
-- ==========================================
CREATE TABLE price_competition (
    id SERIAL PRIMARY KEY,
    item_id VARCHAR(20) NOT NULL,
    snapshot_date DATE NOT NULL,
    -- Preços
    current_price DECIMAL(12,2),
    suggested_price DECIMAL(12,2),              -- Preço sugerido pelo ML
    price_gap DECIMAL(12,2),                    -- current_price - suggested_price
    price_gap_percent DECIMAL(6,2),             -- % acima do sugerido
    -- Status da sugestão
    suggestion_status VARCHAR(30),              -- 'no_benchmark_ok', 'no_benchmark_lowest', etc.
    suggestion_type VARCHAR(30),                -- 'competition' (concorrência) ou 'unhealthy' (campanha)
    -- Tag not_market_price
    has_not_market_price BOOLEAN DEFAULT FALSE, -- Se ML está penalizando
    -- Automatização
    has_price_automation BOOLEAN DEFAULT FALSE, -- Se tem automatização ativa
    automation_rule VARCHAR(20),                 -- 'INT_EXT', 'INT', null
    automation_min DECIMAL(12,2),
    automation_max DECIMAL(12,2),
    -- Buy Box (apenas ~1% catálogo)
    catalog_product_id VARCHAR(30),
    buybox_status VARCHAR(30),                   -- 'winning', 'competing', null (se não catálogo)
    price_to_win DECIMAL(12,2),                  -- Só pra catálogo
    -- Timestamps
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(item_id, snapshot_date)
);

-- ==========================================
-- HISTÓRICO DE MUDANÇAS DE FRETE
-- ==========================================
CREATE TABLE freight_changes (
    id SERIAL PRIMARY KEY,
    item_id VARCHAR(20) NOT NULL,
    change_date TIMESTAMPTZ DEFAULT NOW(),
    old_cost DECIMAL(10,2),
    new_cost DECIMAL(10,2),
    change_percent DECIMAL(6,2),
    free_shipping BOOLEAN,
    auto_removed BOOLEAN DEFAULT FALSE        -- Se foi removido automaticamente
);

-- ==========================================
-- VENDAS DIÁRIAS POR ITEM (gráfico de tendência)
-- Retenção: manter últimos 180 dias, agregar mais antigo por semana
-- Estimativa: ~15.000 itens x 180 dias = ~2.7M linhas (gerenciável no PostgreSQL)
-- ==========================================
CREATE TABLE item_daily_sales (
    id SERIAL PRIMARY KEY,
    item_id VARCHAR(20) NOT NULL,
    sale_date DATE NOT NULL,
    quantity INTEGER DEFAULT 0,
    revenue DECIMAL(12,2) DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(item_id, sale_date)
);

CREATE INDEX idx_daily_sales_item ON item_daily_sales(item_id, sale_date DESC);

-- ==========================================
-- FILA DE CORREÇÕES
-- ==========================================
CREATE TABLE corrections (
    id SERIAL PRIMARY KEY,
    account_id INTEGER REFERENCES accounts(id),
    item_id VARCHAR(20) NOT NULL,
    -- Tipo de correção
    correction_type VARCHAR(50),              -- 'specs_fill', 'specs_hidden', 'title_optimize',
                                              -- 'price_adjust', 'ean_fill', 'compat_fill'
    -- Modo de execução
    execution_mode VARCHAR(20),               -- 'auto', 'approval', 'suggestion'
    -- Rota de envio
    route VARCHAR(20),                        -- 'anymarket', 'ml_direct'
    -- Dados da correção
    field_name VARCHAR(100),                  -- Nome do campo/atributo
    old_value TEXT,
    new_value TEXT,
    correction_data JSONB,                    -- Dados completos da correção
    -- Impacto estimado
    estimated_score_gain DECIMAL(5,2),        -- Quantos pontos ganha
    -- Status
    status VARCHAR(20) DEFAULT 'pending',     -- 'pending', 'approved', 'rejected', 'executed', 'verified', 'failed'
    approved_by VARCHAR(100),
    approved_at TIMESTAMPTZ,
    executed_at TIMESTAMPTZ,
    verified_at TIMESTAMPTZ,                  -- Confirmou no ML que corrigiu
    error_message TEXT,
    -- Timestamps
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_corrections_status ON corrections(status, execution_mode, created_at DESC);
CREATE INDEX idx_corrections_item ON corrections(item_id, status);

-- ==========================================
-- CACHE DE ATRIBUTOS POR CATEGORIA
-- ==========================================
CREATE TABLE category_attributes_cache (
    id SERIAL PRIMARY KEY,
    category_id VARCHAR(20) UNIQUE NOT NULL,
    domain_id VARCHAR(100),
    attributes JSONB,                         -- Resposta completa do /categories/.../technical_specs/input
    total_attributes INTEGER,
    required_count INTEGER,
    hidden_count INTEGER,
    common_count INTEGER,
    cached_at TIMESTAMPTZ DEFAULT NOW()
);

-- ==========================================
-- ÍNDICES
-- ==========================================
CREATE INDEX idx_account_health_date ON account_health(account_id, snapshot_date DESC);
CREATE INDEX idx_items_account_status ON items(account_id, status);
CREATE INDEX idx_items_category ON items(category_id);
CREATE INDEX idx_items_abc ON item_sales(abc_revenue, snapshot_date DESC);
CREATE INDEX idx_item_perf_date ON item_performance(item_id, snapshot_date DESC);
CREATE INDEX idx_item_perf_score ON item_performance(score, snapshot_date DESC);
CREATE INDEX idx_item_exp_gauge ON item_experience(health_gauge, snapshot_date DESC);
CREATE INDEX idx_item_specs_score ON item_specs(score_percent, snapshot_date DESC);
CREATE INDEX idx_alerts_open ON alerts(account_id, resolved, severity, created_at DESC);
CREATE INDEX idx_price_comp_status ON price_competition(suggestion_status, snapshot_date DESC);
CREATE INDEX idx_price_comp_item ON price_competition(item_id, snapshot_date DESC);
CREATE INDEX idx_freight_item ON freight_changes(item_id, change_date DESC);
```

## 5.11 collectors/price_monitor.py (MONITORADOR DE PREÇOS — 99% FORA DO CATÁLOGO)

```python
"""
Monitora competitividade de preço para anúncios TRADICIONAIS (fora do catálogo).

IMPORTANTE: 99% dos anúncios NÃO são de catálogo, então o endpoint
/price_to_win NÃO funciona para a maioria. O monitoramento usa:

Fonte 1 — Sugestões de preço do ML (PRINCIPAL):
  GET /suggestions/user/$USER_ID/items
    → Lista de item_ids que têm sugestão de preço
  GET /suggestions/items/$ITEM_ID/details
    → Preço sugerido baseado em concorrentes internos E externos
    → Status: no_benchmark_ok (preço bom), no_benchmark_lowest (abaixo do sugerido)
    → Tipo de sugestão: concorrência de preços ou campanha unhealthy

  ATENÇÃO: O preço sugerido é INFORMATIVO, nunca automático.
  Muitas vezes o ML compara produtos errados e a sugestão fica discrepante.
  O sistema mostra a sugestão, a equipe decide se aplica ou não.

Fonte 2 — Tag not_market_price (REATIVO):
  Já coletada pelo tags_collector.
  Quando presente, o ML já está PENALIZANDO visibilidade.
  Alerta imediato: 🔴

Fonte 3 — Automatização de preços do ML (OPCIONAL — desativado por padrão):
  GET /pricing-automation/items/$ITEM_ID/automation
    → Verifica se o item tem automatização ativa
    → Se tiver, o preço não pode ser editado via API
  POST /pricing-automation/items/$ITEM_ID/automation
    → Ativa automatização com regra INT_EXT e limites min/max
    → NÃO ativar automaticamente — apenas informar no relatório
    → A decisão de ativar é humana (risco de guerra de preços)

Fonte 4 — price_to_win (SECUNDÁRIO — apenas ~1% catálogo):
  GET /items/$ITEM_ID/price_to_win?version=v2
    → Só funciona pra itens com catalog_product_id
    → Mantido como secundário para os poucos itens de catálogo

Fluxo:
1. GET /suggestions/user/$USER_ID/items → lista de itens com sugestão
2. Para cada item da lista: GET /suggestions/items/$ITEM_ID/details
3. Comparar preço atual vs. preço sugerido
4. Calcular gap: (preço_atual - preço_sugerido) / preço_sugerido * 100
5. Salvar em price_competition
6. Gerar alertas:
   - 🔴 not_market_price detectado (já perdendo visibilidade)
   - 🟡 Preço >15% acima da sugestão ML (risco de tag)
   - ℹ️ Preço sugerido disponível (informativo)

Priorização: Curva A 1x/dia, resto 2x/semana
"""
```

```python
"""
Monitora compatibilidades de veículos para itens de autopeças.
MÓDULO PRINCIPAL — o ML cobra bastante em autopeças, item sem
compatibilidade perde exposição e pode ser pausado.

Segmento: carros e picapes (linha leve). Não inclui motos nem caminhões.
Categoria MLB: MLB1747 (Acessórios para Veículos), MLB22693 (Autopeças)
Domínio: MLB-CARS_AND_VANS

Fluxo de monitoramento:

   GET /users/$SELLER_ID/items/search?tags=incomplete_compatibilities

   GET /users/$SELLER_ID/items/search?tags=pending_compatibilities
     → Itens que TÊM sugestões de compatibilidade do ML
   GET /users/$SELLER_ID/items/search?tags=incomplete_position_compatibilities
     → Itens com posição de aplicação pendente

2. Para cada item: verificar compatibilidades atuais:
   GET /items/$ITEM_ID/compatibilities
     → Lista de veículos compatíveis já cadastrados

3. Buscar sugestões de compatibilidade do ML:
   POST /catalog_compatibilities/products_search/chunks
   {
     "domain_id": "MLB-CARS_AND_VANS",
     "site_id": "MLB",
     "filter": "SUGGESTED",
     "item_id": "$ITEM_ID",
     "known_attributes": [
       {"id": "BRAND", "value_name": "Volkswagen"},
       {"id": "MODEL", "value_name": "Gol"}
     ]
   }
     → Retorna veículos sugeridos pelo ML baseado no produto

4. Contar veículos disponíveis por domínio (validação):
   POST /catalog_compatibilities/products_search/count_family_products
     → Valida antes de associar (máximo 200 produtos por chamada)

5. Para itens universais (servem em qualquer veículo):
   POST /items/$ITEM_ID/compatibilities
   {"products":[], "products_families":[], "products_group":[], "universal": true}

6. Gerar alertas:
   - 🔴 incomplete_compatibilities: item SERÁ pausado se não resolver
   - 🟡 pending_compatibilities: ML tem sugestões, só precisa aprovar
   - 🟡 incomplete_position_compatibilities: falta posição de aplicação
   - ℹ️ Compatibilidade cadastrada mas incompleta (<5 veículos)

Rate limit: 100 rpm por APP_ID para endpoints de compatibilidade
Máximo: 200 produtos por chamada de associação

Salva em: item_specs (campo compatibilities_status) + alerts
Mostra em: tela "Saúde dos anúncios" com filtro por compatibilidade
"""
```
